#### Docker Volume
- 도커는 컨테이너 안의 파일 변경 사항을 UnionFS를 통해 관리
	- Union FS : 이미지 레이어와 write 레이어를 합쳐 컨테이너의 데이터를 관리하는데, 컨테이너 삭제 시 write 레이어도 삭제 된다.
		-> write 레이어에는 이미지 레이어의 데이터에서 변경된 사항을 저장하므로 write 레이어 삭제 시 데이터가 사라진다. ( 휘발성 데이터 )
	- 컨테이너의 데이터 휘발성 때문에 데이터를 컨테이너가 아닌 호스트에 저장할 때 혹은 컨테이너 간 데이터 공유 시 Volume이 필요하다.
