SMP 내부의 여러 기능에서 공통으로 사용할 수 있는 검색 컴포넌트 제작

1. 검색 결과 카운트를 나타낼 수 있으며,
2. 어떤 기능에서도 검색을 수행할 수 있어야 하며,
3. 검색 결과가 각 기능의 테이블에 표시되어야 하고,
4. 검색 초기화 했을 때 원본 배열이 테이블에 표시되어야 한다.

- 어려운 점
	1. 각기 다른 기능에서 한번에 사용할 수 있어야 하기 때문에 기능 별 배열 프로퍼티 값을 어떻게 적용할건지 어려움이 있었음
		-> 사용자에게  직접 값을 받는 식으로 우선 결정
	1. 부모 페이지와 자식 컴포넌트 사이의 상태 관리가 어려움
		- [ ] 전체 페이지에서 표시되는 원본 테이블 내부의 데이터 수와 검색 컴포넌트 내부에서 표현될 검색 결과의 값 데이터가 일치(초기는 전체 데이터, 검색 하면 검색된 데이터의 수)해야 하는데, 이를 일치시키기 쉽지 않음
		- [x] 검색 결과 상태를 부모 컴포넌트로 이동 -> 완료
