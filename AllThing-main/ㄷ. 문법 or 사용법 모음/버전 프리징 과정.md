프로젝트에서 특정 버전을 **freezing (고정, 동결)** 한다는 것은, **해당 버전의 상태를 변경 없이 안정적으로 유지**하고, **참조 기준으로 삼거나 배포 기준점**으로 활용하기 위한 작업이에요.

단순히 코드를 저장하는 것을 넘어서, **누가 봐도 그 버전은 확정되고 더는 바뀌지 않는 상태**를 만들어야 하죠.

---

## ✅ 버전 Freezing을 위한 필수 사항

---

### ✅ 1. **Git 태그(Tag) 생성 (기초 작업)**

bash

복사편집

`git tag v1.0.0 git push origin v1.0.0`

- 커밋이 정확히 고정됨
    
- 변경 불가능한 참조점으로 활용 가능
    

---

### ✅ 2. **GitLab 릴리즈(Release) 작성**

- 해당 태그에 대한 설명, 배포 내용 등을 포함한 릴리즈 작성
    
- UI 또는 GitLab API를 통해 생성
    
- `.zip`, `.exe`, `.apk` 등 빌드 결과물 첨부 가능
    

---

### ✅ 3. **브랜치 보호 (Protect Tag / Branch)**

- GitLab의 **Protected Tags** 또는 **Protected Branches** 설정
    
- `v1.0.0` 태그가 삭제되거나 덮어쓰기(push -f) 되지 않도록 막기
    

#### 🔧 GitLab 설정 방법:

> Project → Settings → Repository → Protected Tags

- `v*` 패턴을 보호하고, 관리자 외에는 수정 불가하게 설정
    

---

### ✅ 4. **종속성 버전 고정 (Dependency Freeze)**

- Node.js: `package-lock.json` 또는 `yarn.lock`
    
- Python: `requirements.txt`에 버전 명시 (`fastapi==0.100.0`)
    
- Java: `pom.xml`에 버전 고정
    
- Docker: `FROM node:18.16.0` 식으로 버전 고정
    

📌 코드만 Freezing하면 안 되고, **외부 의존성도 동일하게 유지되도록** 관리해야 함

---

### ✅ 5. **CI/CD 환경에서 해시/태그 고정**

- 특정 배포/빌드는 `v1.0.0` 태그를 기준으로 실행되도록
    

yaml

복사편집

`# 예: GitLab CI only:   - tags`

- 또는 특정 커밋 해시 기반으로만 동작하게 설정 (`commit SHA`)
    

---

### ✅ 6. **Artifact 백업 및 배포 스냅샷 보관**

- 빌드 결과물(zip, apk 등)을 S3, GitLab Releases, 사내 서버 등에 보관
    
- Docker 이미지도 태그 기반으로 보관 (`myapp:1.0.0`)
    

---

### ✅ 7. **변경 금지 원칙 명시 (README, 문서)**

- `v1.0.0`은 변경 불가, 추가 수정은 `v1.0.1` 이상에서 브랜치 새로 따서 진행
    
- 예시 문구:
    
    > 🔒 `v1.0.0`은 릴리즈된 상태로 동결되었으며, 이후 수정은 `v1.0.x` 브랜치에서 관리합니다.
    

---

## 🧊 Freezing 체크리스트 요약

|항목|설명|
|---|---|
|Git 태그 생성|커밋을 고정|
|릴리즈 작성|설명, 아티팩트 포함|
|Protected Tags 설정|덮어쓰기/삭제 방지|
|종속성 버전 고정|lock 파일 또는 명시적 버전|
|CI/CD에 태그 기반 배포|항상 동일 결과 보장|
|빌드 파일 보관|zip, 이미지, 아카이브 등|
|문서화|Freezing 정책 명시|

bash

`# Git 태그 삭제 방지 git config --local receive.denyDeleteTag true`
