
#### Server
- **proxy**
	- **rewrite** : Vite 프록시 서버에서 실제 API 서버로 전달하는 URL을 수정할 수 있도록 한다.
		- 서버의 요청 경로를 변경하지만, 브라우저에서 네트워크 탭에 표시되는 url은 여전히 클라이언트 측에서 보낸 원본 URL을 반영한다.
		- 동작 흐름
			1. 클라이언트 측에서 변경되지 않은 URL로 요청
			2. Vite 개발 서버는 이 요청을 프록시 서버로 전달하기 전에 rewrite 옵션을 사용해 URL을 수정한다.
			3. 프록시 서버는 실제로 수정된 URL을 받는다.
			4. 이 후 프록시 서버는 수정된 URL을 실제 API 서버로 전달한다.
			5. 실제 API 서버는 변경된 URL 경로에 대한 요청을 처리한다.
