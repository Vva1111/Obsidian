## 개요
- 기존 메인 프로젝트(BMP) 와는 별개로 수집 대상이 다르고, 제공 대상이 다른 프로그램을 제공하기 위해 CMP(Spinel) 기능을 추가해 완전 새로운 프로젝트를 하나 생성해야 하고, 기존 프로젝트(BMP)도 유지보수를 위해 함께 관리가 필요한 상황
- Turborepo tool 사용

## 과정
- 기존 A 프로젝트를 바탕으로 turborepo 를 설치해서 구조 적용할 예정이었으나, 오류가 많아 최초 turbrepo 를 사용해 프로젝트 생성 후 apps 내부에 각 A, B 각 프로젝트를 가져와서 적용하는 방식으로 변경

## 실제 적용
- 실제 프로젝트에는 적용하지 않기로 결정
	1. 사용 방식이나 구조 변경으로 개발자 혼동 우려
	2. 모노레포로 변경하면서 추후 발생하게 될 사이드 이펙트
	3. 위 사항 관련 유지보수 기간이 개발 기간내에 어려움
	4. 개인적으로 공통 라우팅이나 유팅 사용 추가 개발
