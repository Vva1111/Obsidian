```ts
import { defineConfig } from 'vite';

export default defineConfig({
  build: {
    rollupOptions: {
      input: {
        main: 'src/main.js',
        // production 환경에서 제외할 파일
        ...(process.env.NODE_ENV === 'production' 
								        ? {} 
										: { exclude: 'src/파일명.js' }),
      },
    },
  },
});
```
- **Vite.config rollupOptions 내부 input 옵션에 대해
	- Rollup 빌드 프로세스에서 사용할 진입점 파일을 정의한다.
	- Vite는 내부적으로 Rollup을 사용해 애플리케이션을 번들링하므로, 이 옵션은 Rollup의 설정과 밀접한 연관이 있다.
	
- **Input 사용 예시
	- 단일, 다중 진입점을 정의할 수 있으며, 단일은 string 형태로, 다중은 객체 형태로 정의한다.
	```ts
	input: {
	  main: 'src/main.js',
	  admin: 'src/admin.js',
	}
	```
	- 위 예제에서는 main이 src/main.js로 설정되어 있고, 이는 Vite가 애플리케이션을 빌드할 때 src/main.js 파일을 시작점으로 사용한다는 의미이다.

- **Input 옵션 사용의 이점
	- **모듈화**
		진입점 설정으로 애플리케이션을 여러 개의 모듈로 나눌 수 있다. 각 모듈은 독립적으로 개발 및 테스트할 수 있으며, 이는 코드 유지보수성을 높일 수 있다.
		
	- **코드 분할**
		Rollup은 진입점을 기반으로 코드 분할을 수행할 수 있다. 이를 통해 필요한 모듈만 로드해 초기 로딩 시간을 줄일 수 있다.
		
	- **다중 페이지 애플리케이션**
		여러 개의 진입점을 설정하면 다중 페이지 애플리케이션을 쉽게 구축할 수 있다.
		각 페이지는 별도의 진입점을 가질 수 있으며, 이를 통해 필요한 리소스만 업로드할 수 있다.
		
	- **컴포넌트 라이브러리**
		컴포넌트 라이브러리를 개발할 때, 각 컴포넌트를 별도의 진입점으로 설정해 개별적으로 빌드하고 배포할 수 있다.
		
	- **환경별 설정**
		진입점을 설정하면 개발 환경과 프로덕션 환경에서 서로 다른 파일을 포함하거나 제외할 수 있다.
