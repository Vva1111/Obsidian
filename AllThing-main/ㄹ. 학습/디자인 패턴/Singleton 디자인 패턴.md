- 어떤 클래스의 인스턴스를 하나만 생성하고, 전역에서 그 인스턴스에 접근하도록 보장하는 패턴
- 전역으로 공유되는 자원(설정, 로거, 캐시, DB 커넥션 등)을 단일 인스턴스로 관리
- 인스턴스 생성을 통제하고, 동일한 상태/동작을 모든 호출자가 공유하도록 함

### 개념적 구성 요소
- Singleton 클래스 - 생성자(외부에서 직접 생성 불가), 인스턴스를 반환하는 정적 접근자(getInstance() 등).
- 내부 저장소 - 단 하나의 인스턴스를 보관하는 정적/모듈 수준 변수
- (선택) 초기화 로직 : 지연 초기화(lazy) 또는 즉시 초기화(eager)

### 사용
- 전역 설정(config) 객체
- 애플리케이션 수준 로거
- 커넥션 풀 / 데이터베이스 클라이언트 ( 주의 필요 )
- 애플리케이션 전체에서 공유되는 캐시(읽기 중심)
- 리소스가 비용이 크고 하나만 있으면 되는 경우
