- docker run -it "test" centos /bin/bash 명령
	- centOS 이미지를 사용해 새로운 컨테이너를 생성하고, 그 컨테이너 내부에서 /bin/bash shell을 실행하게 하는 명령
	- -it : 인터랙티브 모드 실행 명령, 사용자가 컨테이너 shell에 직접 접근할 수 있도록 한다.
	
	- 위 명령을 실행하면,
		1. 프로세스 종료 : /bin/bash shell이 실행된 후, 사용자가 shell을 종료하면 해당 프로세스 또한 종료된다. 
		   Docker 컨테이너는 기본적으로 실행 중인 프로세스가 없을 경우, 컨테이너가 종료된다. 따라서, 위 명령 실행 후 shell 을 종료하면 해당 컨테이너 또한 stop 상태가 된다
		   
		2. docker run : docker run 명령어는 지정된 명령어가 실행되고, 종료될 때 까지만 컨테이너를 실행한다. 따라서, /bin/bash 가 종료되면 컨테이너도 종료된다.

- 프롬프트(ex. ubuntu@instance-20240000:~$) : 
	사용자 이름@호스트 이름 : 작업 디렉토리(~) 사용자 권한(\$)

- docker run -d --name "centos-test" centos /bin/ping localhost
	- centos 이미지를 사용해 /bin/ping shell 명령을 localhost 대상으로 사용하는 'centos-test' 라는 이름의 컨테이너를 생성한 후, background 실행

- docker run -d --name "webtest" -p 8080:80 nginx
	- nginx 이미지를 사용해 호스트의 8080 포트와 매핑되는 80번 포트가 설정된 "webtest"라는 이름의 컨테이너를 생성한 후, background 실행
	- 해당 명령을 실행하면, 호스트의 8080번 포트로 접근할 경우, 컨테이너에서 작동하고 있는 nginx(80) 서비스에 접근할 수 있다.

- docker run -v 호스트디렉토리:컨테이너디렉토리 ~~ 를 사용해서 호스트-컨테이너 디렉토리를 공유할 수 있다.
- docker run 실행 시, 환경 변수를 지정할 수 있다.

<h4>Docker Network</h4>
- Docker 는 기본 네트워크로 bridge, host, none 세 개의 네트워크를 만든다.

- 네트워크를 명시적으로 지정하지 않고 Docker 컨테이너를 시작하면 기본 값인 'bridge' 네트워크로 docker 컨테이너를 시작한다.
	- 오버레이 네트워크 :
		- 물리 네트워크 상에서 소프트웨어적으로 에뮬레이트 한 네트워크를 뜻하며, 물리 네트워크를 덮듯이 가상 네트워크가 구성된다는 점에서 가상 네트워크라고도 한다.
		- 물리 네트워크의 구조가 은폐되어 그 아래에 있는 물리 계층의 형태나 제어 방식 등을 의식하지 않고 사용할 수 있다. 예를 들어, 여러 개의 호스트에 걸친 네트워크를 구성할 때 사용한다.

- Docker의 기본 bridge 네트워크와 사용자 정의(docker network create) 방식으로 임의로 만들어진 네트워크에서는 이름 해결 구조가 다르다.
	- 컨테이너 실행시, 네트워크를 명시하지 않을 경우, 이름을 해결할 수 없으므로 docker run 명령으로 --link 옵션을 붙여둘 필요가 있다.
	  --link 옵션을 지정할 경우, 컨테이너 내부의 /etc/hosts 파일에 컨테이너의 이름과 컨테이너에 할당된 IP 주소가 등록된다.
	- 사용자 정의 네트워크는 docker 데몬에 내장된 내부 DNS 서버에 의해 이름 해결이 이루어진다.
	  내부 DNS 서버를 사용하면 link 기능과 같이 /etc/hosts 에 의존하지 않고 이름 해결이 가능하다. 그렇기 때문에 사용자 지정 네트워크를 사용하는게 유연하고 쉽게 네트워크 구성이 가능하다.
	  
- docker connect(or disconnect) <network 이름> <컨테이너 이름>
	- 위 명령을 사용하면, 해당하는 네트워크에 컨테이너를 연결하거나, 연결을 해제할 수 있다.
	- 연결 후에는 동일한 네트워크 상에 있는 다른 컨테이너와 통신할 수 있다.
	- 각 컨테이너의 연결은 IP, 컨테이너 이름 또는 컨테이너 ID로 가능하다.
	
- docker run 명령과 함께 네트워크 정보 구성도 가능하다.
- docker network create 명령으로 사용자 정의 네트워크를 구성할 수 있다.
- 브릿지 네트워크 : 
	- Docker 에서 컨테이너 사이 통신을 가능하게 하는 가장 기본적인 네트워크 유형 중 하나이다. 
	- Docker 가 기본적으로 제공하는 가상 네트워크로, 여러 컨테이너가 동일한 네트워크에 연결되어 서로 통신할 수 있도록 한다.
