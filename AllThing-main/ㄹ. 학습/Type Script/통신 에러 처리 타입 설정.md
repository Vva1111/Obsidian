#### api 통신의 예외 처리를 위한 try, catch 문에서 catch의 error인자에 대한 타입 설정을 하는 방식에 대한 설명

![[통신 에러 처리(catch) 에서 타입 설정 방법.png]]

- error의 타입은 unknown 혹은 any만을 사용하게 되는데, any[^1]의 사용은 가급적 지양하는 편이다.
- unknown 타입은 any보다 엄격한 타입으로, 어떤 값이든 할당할 수 있지만 사용하기 전에 타입을 확인하거나 변환해야 한다. 이 타입의 값은 일반적으로 어떤 값도 수행할 수 없지만 타입 검사기가 unknown 타입의 값에 대한 작업을 허용하는 경우, 해당 값이 사용될 때 타입 안정성을 보장한다. - ChatGPT
- 위의 이유로 error 값을 unknown으로 잡고, 조건문으로 에러의 타입을 확정시키는 방법으로 사용해야 한다.

---
[1.]: any의 경우, 타입 검사를 우회하는 것으로 간주하며, 어떤 값이든 할당 가능하고 어떤 연산도 가능하게 한다. 이는 타입 안정성을 보장하지 않는 것이다.
