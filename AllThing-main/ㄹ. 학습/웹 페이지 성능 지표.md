
#### LCP(Large Contentful Paint) : 
- 사용자가 페이지를 방문하고 그 페이지의 주요 콘텐츠가 얼마나 빠르게 화면에 그려지는지 측정하는 지표
- 브라우저가 렌더링하는 뷰포트 내에서 가장 큰 영역을 차지하는 요소(이미지, 비디오 포스터 이미지, 블록 레벨의 텍스트 요소 등) 가 완전히 로드되는 시점을 측정한다.
- 이 시점은 페이지 로드 시간과 사용자의 첫 인상에 큰 영향을 미친다.
- 이 지표가 낮을수록 사용자 경험이 좋다고 평가할 수 있으며, 구글은 2.5초 이하 유지 권장

#### FCP(First Contentful Paint) : 
- 웹 사이트를 방문한 후 브라우저가 페이지의 첫 번째 콘텐츠를 그리기 시작하는 시점을 나타내는 지표
- 이 콘텐츠는 텍스트, 이미지, SVG 등이 될 수 있음
- 이 지표는 웹 사이트의 초기 로딩 속도를 나타내며, 이 시점이 빠를수록 사용자 경험이 좋다고 평가

#### TTI(Time To Interactive)
- 웹 페이지가 완전히 로드되어 사용자의 상호작용(클릭, 스크롤, 키보드 입력 등)에 반응할 준비가 된 시점을 의미하는 지표
- 단순히 페이지가 보이는 것이 아닌, 실제로 사용자의 입력에 반응할 수 있는 상태가 되는 시점을 의미

#### TTFI(Time To First Interactive)
- TTI 와 유사하지만, 페이지가 실제로 상호작용 가능한 상태가 되는 시점을 측정한다.
- TTI는 페이지가 안정적으로 상호작용 가능한 상태가 되는 시점을 측정하는 반면, TTFI는 페이지가 처음으로 상호작용 가능한 상태가 되는 시점을 측정한다.

#### TTFB(Time To First Byte)
- 웹 브라우저가 첫 번째 바이트를 서버로부터 받는 데 걸리는 시간을 측정하는 지표, 서버의 응답 시간을 나타낸다

#### FID(First Input Delay)
- 사용자가 웹 페이지에 처음으로 상호작용(클릭, 탭, 키보드 입력 등)을 시작한 후, 브라우저가 이를 처리하는데 걸리는 시간을 측정하는 지표
- 이 지표는 웹사이트가 사용자의 상호작용에 얼마나 빠르게 반응하는지 나타낸다.

#### CLS(Cumulative Layout Shift)
- 웹 페이지의 레이아웃이 얼마나 많이 변하는지 측정하는 지표
- 페이지의 안정성을 나타내는 지표
- 이동하는 이미지, 광고, 동적 콘텐츠 등으로 인해 레이아웃이 자주 변경되면 사용자 경험이 좋지 않을 수 있다.
#### Speed Index
- 페이지의 시각적 요소가 얼마나 빠르게 로딩 되는지 측정하는 지표
- 사용자가 페이지를 빠르게 볼 수 있는 정도를 나타내기 때문에 사용자 경험과 밀접한 연관이 있다.

#### TBT(Total Blocking Time)
- 페이지가 완전히 상호작용 가능해지기까지의 시간을 측정하는 지표
- 페이지가 사용자의 입력에 즉시 반응하지 못하는 시간을 나타낸다.
